나의 Backend 설계 & 문제 해결

AI 의류 추천 서비스가 안정적으로 동작하도록 인증 플로우, 의류 등록 도메인, DB 스키마 확장을 중심으로 백엔드 구조를 설계하고 개선했습니다.

인증 플로우 안정화
  • JWT 기반 액세스/리프레시 토큰 구조에 이메일 인증, Google OAuth 2.0을 통합해 단일 플로우로 설계
  • 미들웨어로 권한 검증을 공통화하여 신규 API 추가 시 인증 로직 중복과 누락을 방지
의류 등록 도메인 설계
  • 의류 생성/조회 API를 Controller–Service–Repository 계층으로 분리해 유지보수성과 테스트 용이성 확보
  • 카테고리·색상·계절감 등 Vision 응답을 도메인 모델에 맞게 정규화하여 데이터 일관성 강화
Prisma 스키마 확장
  • 이메일 인증, Google Auth, FCM 토큰 등 신규 요구사항을 Prisma 스키마·마이그레이션으로 안전하게 반영
  • seed 스크립트로 초기 데이터 세팅을 자동화하고 로컬/운영 환경 간 동작을 검증
